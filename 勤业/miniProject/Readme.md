# miniProject

> 设计相关迷你项目，为学生提供一些实践机会，以学生**自主推动**为主，会根据学生的进展进行不同程度的指导。



## 开发规则

- 如果对某一个miniProject感兴趣，可直接在个人Gitee（如使用其它平台，可单独与指导教师联系）中新建项目（项目名：miniProject-6或miniProject-10等），并将指导教师 (guangyou_duan)加入自己的项目中；
- 指导教师进入相应项目后联系项目所有者，详细介绍项目、进行初步指导；
- 项目开展前、中、后，可随时通过邮件等与指导教师联系；
- 指导教师会根据相应项目的需要，组建团队，miniProject以个人为主；
- miniProject项目指导老师也会同步独立推动；
- 项目也可以随时中止（不用和指导老师说），项目长期不更新就说明不再推动了；
- 个人新建的miniProject-x，一定要注意权限，除了指导老师外，不允许对外公开；
- 因为涉及保密，项目名称可能过于简单、但会尽量表达清楚方向；
- 项目代码和数据等，每天只要推动，及时提交、更新，便于随时指导；
- 相关文档等写作用markdown格式，参考文献插入时用"第一作者-given-name\_年份\_期刊"指代，相应参考文献放在项目Doc目录，如“Yuumi_2024_Frontiers-in-Immunology.pdf”；
- 其它未尽事宜，会不断更新，也可随时交流。



## 潜在收获

- 有指导情形下独自（协作）参与项目实践。
- 根据参与和推动程度，获得相应补助。
- 获得参与其它研发项目等机会。
- 如果参与的项目，获得论文发表等成果形式，将给予一定额度的奖励。

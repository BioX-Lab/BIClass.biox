# 在BASH中获得上一条命令的执行状态用于控制后续命令的执行

在BASH中，`$?`是一个特殊的变量，用于存储上一个命令的退出状态码。退出状态码是一个整数值，范围是0 - 255，它表示命令执行后的结果状态。
## 状态码含义
   - **0表示成功**：当一个命令成功完成它的任务时，通常会返回0作为退出状态码。例如，`ls`命令在成功列出目录内容后返回0。
   - **非0表示失败或异常情况**：不同的非0值可以表示不同类型的错误。比如，在一些情况下，1可能表示通用错误，2可能表示命令使用不当（如错误的参数），127可能表示找不到命令（当试图执行一个不存在的可执行文件时）。不过，具体的非0状态码的含义可能因命令而异。
## 示例 1
假设你有一个简单的脚本，先尝试执行一个可能不存在的命令，然后检查`$?`的值。
``` bash
#!/usr/bin/bash
non_existent_command
echo "The exit status of the previous command is: $?"
```
   在这个示例中，因为`non_existent_command`不存在，所以会返回一个非0的状态码（很可能是127），`echo`命令会输出这个非0的状态码，显示命令执行出现问题。
## 示例 2

检查一个文件是否存在的`test`命令（`[ ]`是`test`命令的一种语法形式）。
```bash
#!/usr/bin/bash
file="example.txt"
[ -e $file ]
echo "The exit status of the file existence check is: $?"
```
   如果`example.txt`存在，`[ -e $file ]`会返回0，`echo`命令输出0，表示文件存在检查成功；如果文件不存在，`[ -e $file ]`返回非0值，`echo`命令输出这个非0值，表示文件不存在。

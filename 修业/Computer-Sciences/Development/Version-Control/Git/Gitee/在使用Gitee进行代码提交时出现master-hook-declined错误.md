# 在使用Gitee进行代码提交时出现master (hook declined)错误

## 错误描述

> 本问题由Zhaohui.Li/BI2023@QLNU提出。

在尝试将代码推送到Gitee远程仓库时，遇到了`! [remote rejected] master -> master (hook declined)`的错误。这个错误通常发生在远程仓库的“pre-receive hook”阶段，表示远程仓库拒绝了推送操作。

## 可能原因
1. **邮箱设置问题**：用户的邮箱可能被设置为私有，导致无法通过验证。
2. **远程地址问题**：本地仓库的远程地址可能未正确设置或格式不正确。
3. **分支保护问题**：远程仓库可能对`master`分支设置了保护，限制了推送权限。
4. **其他配置问题**：如SSH密钥未正确配置等。

## 解决方案
1. **检查邮箱设置**：
   - 进入Gitee的个人设置，找到邮箱管理，取消“禁止命令行推送暴露邮箱”的选项。(这种情况出现概率较大)

2. **正确设置远程地址**：
   - 使用`git remote add origin "远程地址"`命令正确设置远程地址。
   - 如果已经设置，可以使用`git remote -v`查看当前的远程地址，确保其正确。

3. **解决分支保护问题**：
   - 如果`master`分支被保护，需要项目管理员或有权限的用户解除保护。
   - 在Gitee的项目设置中，找到“Protected Branches”，调整`master`分支的保护设置。

4. **其他配置检查**：
   - 确保SSH密钥已正确配置，特别是如果使用SSH方式连接远程仓库。
   - 如果使用HTTPS方式，确保用户名和密码正确。

## 提交代码的正确流程
> 推荐使用TortoiseGit等界面工具操作。
1. **初始化本地项目**：
   ```bash
   git init
   ```
2. **添加跟踪文件**：
   ```bash
   git add .
   ```
3. **提交到本地仓库**：
   ```bash
   git commit -m '提交的备注'
   ```
4. **连接本地仓库和远程仓库**：
   ```bash
   git remote add origin "远程地址"
   ```
5. **合并远程代码与本地代码**：
   ```bash
   git pull --rebase origin master
   ```
6. **提交到远程仓库**：
   ```bash
   git push -u origin master -f
   ```

## 其他注意事项
- 如果问题仍然存在，可以尝试删除本地的`.git`文件夹，重新初始化仓库并重复上述步骤。
- 确保在提交代码时，本地仓库的分支名称与远程仓库的分支名称一致。

# Git使用规范

关于Git的使用，有几个原则或规范，在实践过程中要尽量去遵循、并逐渐内化成自己的习惯。

- 每一个文件夹下都有一个Readme.md文档，用于说明本文件夹的作用以及里面包含的内容；文件夹内容变动时，及时更新Readme文件。
- 每天及时进行git commit，不用太频繁，但要养成多提交的习惯，以实际进展为准。
- 对于团队协作的项目，个人每次在开发前，先git pull一下，把别人的贡献在本地进行同步，同样在进行git commit或者push前，先git pull一下。
- 每次进行git commit时message要尽可能描述清楚，标注好做了哪些更新，语言简练即可。
- 每次git commit时在message前面加上“[#task-id]”，便于后续统计、代码审阅等。
- “task-id”一般指的是项目规划阶段提前创建好的task (在Gitee中可以用Issues代替，在实际开发场景中，经常会使用[Jira](https://www.atlassian.com/software/jira)等工具对项目和相应的task进行管理）。
- 在研发过程中首先根据项目需要建立对应的issue（或task，以下两者等同），每个issue都会分配给不同的开发者（单人项目只有开发者个人）。
- 在开发过程中，如果没有相应的task (issue)，不建议提交代码，除非项目和相关的任务简单、明确或者处于项目的初始阶段（这里也反映了一个项目中前期的规划和方案设计的重要性，相对来说真正的开发并不是最主要的）。
- 每一个issue在实际过程中遇到的问题，可以随时在对应issue下面进行讨论。

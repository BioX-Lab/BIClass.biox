# PubChem中Canonical与Isomeric类型的SMILES

在PubChem等数据库中，同时提供这两种类型的SMILES，以满足不同用户对于分子结构表示的需求。

## Canonical SMILES（规范SMILES)
这是一种标准化的SMILES表示形式。它通过一套特定的算法规则将分子结构转化为唯一的字符串表示。这种唯一性主要是为了便于在数据库（如PubChem）中进行比较和搜索。例如，对于同一分子的不同输入顺序或表示方式，Canonical SMILES会将它们统一成一种标准形式。
## Isomeric SMILES（异构SMILES）
它不仅包含分子的连接信息，还包含立体化学和同位素等异构信息。这意味着Isomeric SMILES能够更详细地描述分子的结构，包括手性中心、双键的立体化学（如顺反异构）等信息。

## 比较
Canonical SMILES更侧重于提供一种统一的、便于数据库管理和检索的分子表示。它忽略了一些分子的详细异构信息，主要关注原子之间的连接关系。而Isomeric SMILES则是对分子结构的更完整描述，能够准确区分不同异构体。
## 应用场景
- 在简单的分子结构搜索和比较（不涉及异构体区分）时，Canonical SMILES很有用。例如，在一个大型化合物数据库中，快速查找具有相同原子连接方式的化合物，使用Canonical SMILES可以提高效率。
- Isomeric SMILES在涉及到对分子的精确结构分析，尤其是异构体研究（如药物化学中手性药物的不同异构体的性质研究）等场景中发挥关键作用。它可以准确地表示和区分不同的异构体，有助于更深入地了解分子的物理化学性质。
